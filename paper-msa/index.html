💡 AI가 글을 읽는 방법

근데 문제가 있어. 단어가 늘어날수록 연결해야 할 경우의 수가 제곱으로 늘어나거든!

지금까지 다른 사람들은 어떻게 했어?

방법 비유 좋은 점 나쁜 점
Full Attention
(전부 보기)		 📚 모든 책을 다 읽고 기억함 정확도 최고! 메모리 폭발, 너무 느림
RNN
(요약하기)		 🐟 금붕어 기억력처럼 계속 덮어씀 빠름, 메모리 절약 오래된 건 다 잊어버림
RAG
(도서관 검색)		 📖 필요할 때 도서관에서 찾아봄 용량 무제한 검색이 부정확함,

🌟 MSA
(이 논문!)		 🧠 현명한 사서: 관련된 것만 골라서 깊이 읽음 용량 ✓ 정확도 ✓ 속도 ✓ —

🔍 RAG가 뭔지 더 자세히 — 왜 부족했을까?

RAG는 이렇게 작동해:

  1. 질문을 받으면 키워드로 도서관에서 관련 책 찾기
  2. 그 부분만 읽어서 답하기

문제는 "영희는 왜 울었을까?" 같은 질문이야. 영희가 운 이유가 3챕터에 있고, 그 원인은 1챕터에 있을 수 있어. 키워드 검색만으로는 이런 여러 단계를 거치는 추론이 어려워!

Chapter 3

MSA는 어떻게 작동해?

"4가지 핵심 기술이 조화를 이루고 있어"

1
🔦
Memory Sparse Attention
핵심 기억 선택
필요한 기억만 쏙쏙 골라서 집중하는 방법
🔦 손전등 사서 비유:
도서관에 책이 100만 권 있어. 사서가 모든 책을 다 보는 게 아니라, 질문과 관련된 책 표지(라우팅 키)만 먼저 쭉 훑어보고, 관련 있는 책 몇 권만 꺼내서 자세히 읽는 거야!

기술적으로는 두 단계 라우팅을 써:
1단계: 주제별로 대충 훑기 (청크 풀링)
2단계: 관련 청크 안에서 세밀하게 선택
2
📍
Document-wise RoPE
위치 혼동 방지
각 문서마다 자기만의 페이지 번호를 쓰는 방법
📍 각자 페이지 번호 비유:
국어 교과서는 1~300페이지, 수학 교과서는 1~200페이지. 각각 자기 번호를 써. 만약 전교생 교과서 번호를 하나로 합치면 혼돈이 오겠지?

MSA는 각 문서가 항상 1페이지부터 시작하게 해서, 문서가 몇 번째 위치에 있든 혼동이 없어. 이 덕분에 짧은 글로 훈련해도 긴 글에서 잘 동작해!
3
🗜️
KV 캐시 압축 + 병렬처리
메모리 절약 & 속도
기억을 압축하고 두 컴퓨터로 나눠서 처리하는 방법
🗜️ 진공포장 비유:
이불 100개를 보관해야 해. 그냥 쌓으면 방이 꽉 차. 진공포장으로 꾹꾹 누르면 공간 절약!

MSA는 기억(KV 캐시)을 이렇게 나눠:
• 빠른 기억(GPU): 어떤 문서가 관련 있는지 판단하는 키
• 느린 기억(CPU RAM): 실제 내용은 필요할 때만 가져옴

이렇게 해서 A800 GPU 2대만으로 1억 토큰 처리 가능!
4
🔄
Memory Interleave
반복 추론
여러 단서를 오가며 복잡한 문제를 푸는 방법
🔄 추리 소설 비유:
탐정이 범인을 찾으려면:
1. 1번 단서 → "범인은 키가 크다"
2. 다시 도서관 → 키 큰 사람 목록 검색
3. 3번 단서 → "오른손잡이"
4. 다시 좁히기 → 최종 범인!

이렇게 여러 번 왕복하면서 복잡한 다단계 질문에 답해. 이걸 제거하면 성능이 5.3% 하락하고, 복잡한 질문에서는 19%나 떨어져!

🔧 왜 기존 RAG보다 똑똑할까?

RAG는 AI와 검색이 따로따로 동작해. 검색 결과가 틀려도 AI가 수정할 수 없어.

MSA는 검색과 AI가 같이 훈련돼. 어떤 문서를 선택할지도 AI가 직접 배워서, 훨씬 정확한 문서를 골라낼 수 있어!

🔎 두 단계 선택 — 어떻게 관련 문서만 골라낼까?

MSA의 핵심 중 핵심인 듀얼 라우팅(Dual Routing)을 좀 더 자세히 살펴보자.

🏫 도서관 사서 두 명 비유

엄청나게 큰 도서관(1억 토큰 = 책 수백만 권)에서 "조선 시대 임금님 밥상이 궁금해!" 라고 질문했어.

  • 1단계 사서 (주제 훑기): 수백만 권 중에서 표지와 목차만 쓱 훑어봐. "역사", "조선", "음식" 이런 주제가 맞는 책장(청크) 수백 개를 추려.
  • 2단계 사서 (세밀 선택): 추려진 책장 안에서 실제 내용을 읽어봐. 딱 맞는 페이지 64개짜리 묶음(청크)만 최종 선택해.

이렇게 두 번 걸러내기 덕분에 1억 개 중에서도 진짜 필요한 부분만 빠르게 찾을 수 있어!

📐 청크(Chunk)가 뭐야?

MSA는 긴 문서를 64토큰짜리 덩어리(청크)로 잘라서 저장해. 마치 책을 페이지 단위로 자른 것처럼.

1단계에서는 각 청크를 하나의 "대표값"으로 요약해서 훑고, 관련 있는 청크만 2단계로 보내. 이 덕분에 전체를 다 읽지 않아도 돼!

🚫 쏠림 방지 장치 (Auxiliary Loss)

문제가 생길 수 있어. AI가 게을러져서 항상 같은 인기 있는 문서만 고르면 어떡할까?

MSA는 훈련할 때 보조 점수(L_aux)를 계산해서, "너무 한쪽 문서에만 집중하면 감점!"을 줘. 덕분에 AI가 골고루 다양한 문서를 정확히 선택하도록 배워.

이걸 제거하면 성능이 무려 31% 하락해 — 그만큼 중요한 장치야.

Chapter 3.5

MSA는 어떻게 배웠어?

"똑똑해지려면 특별한 훈련이 필요해!"

MSA가 처음부터 1억 토큰을 잘 다루도록 훈련된 건 아니야. 두 단계 훈련을 거쳤어.

🌱
1단계: 기반 만들기
Qwen3-4B라는 기존 AI를 출발점으로 삼아. 처음부터 만들지 않아도 돼!
→
🏋️
2단계: 지속 사전학습
라우팅 능력을 집중 훈련. "어떤 문서를 골라야 해?" 를 수없이 반복 연습.
→
🎓
3단계: 파인튜닝
실제 질문-답변 데이터로 마무리 연습. 실전 능력 향상!

🏊 수영 선수 비유

  • 1단계: 이미 수영을 잘하는 선수를 데려옴 (기존 AI 활용)
  • 2단계: "빠른 코스 파악" 능력만 집중 훈련 (라우팅 특화 학습)
  • 3단계: 실제 대회 규칙에 맞게 연습 (파인튜닝)
💡 짧게 훈련해도 길게 쓸 수 있는 이유

MSA는 64K 토큰(소설 약 0.2권) 분량으로만 훈련했어. 근데 실제로는 1억 토큰(소설 333권)에서도 잘 돼!

비결은 Document-wise RoPE야. 각 문서가 항상 자기 자신의 1페이지부터 시작하니까, AI 입장에서는 긴 문서를 읽는 게 아니라 짧은 문서를 여러 개 읽는 것처럼 느껴지거든. 그래서 훈련 범위를 넘어선 길이도 자연스럽게 처리할 수 있어!

🔌 기존 AI에 꽂아 쓸 수 있어 (플러그인 방식)

MSA 레이어는 기존 트랜스포머 블록에서 셀프 어텐션 레이어만 교체하는 방식이야.

마치 컴퓨터 부품을 업그레이드하듯, 기존 AI 구조를 통째로 바꾸지 않고도 MSA를 끼워 넣을 수 있어. 이미 잘 만들어진 AI들에 기억력 업그레이드를 추가하는 게 가능하다는 뜻!

🚀
MSA가 열어주는 미래 지금은 AI한테 긴 대화를 하면 앞 내용을 잊어버려. MSA 덕분에 앞으로는:
  • 📖 회사 문서 전체를 기억하는 업무 AI
  • 🏥 환자의 평생 의료 기록을 분석하는 의료 AI

  • 🎮 수십 년의 게임 역사를 기억하는 AI
🔮 Memory-as-a-Service (서비스로서의 기억)

MSA의 연구팀은 미래에 "기억"이 하나의 독립된 서비스가 될 거라고 해.

마치 클라우드 저장소처럼, AI 기억도 어떤 AI 엔진에나 꽂아서 쓸 수 있게 된다는 거야. 그러면 네 데이터가 하나의 AI 회사에 갇히지 않아도 돼!

📚 용어사전

어려운 말 쉽게!

토큰 (Token)
AI가 글을 읽는 최소 단위. 대략 단어 하나~두 개에 해당해. "안녕하세요"는 약 4~5개 토큰.
어텐션 (Attention)
AI가 글을 읽을 때 "이 단어와 저 단어가 얼마나 관련 있어?" 하고 연결 강도를 계산하는 방법.
KV 캐시
AI가 읽은 내용을 임시 저장하는 공간. Key(색인)와 Value(내용)로 나뉘어 저장해.
RAG
Retrieval-Augmented Generation. AI가 외부 데이터베이스에서 관련 정보를 검색해서 답하는 방법. 도서관에 가서 찾아보는 것과 비슷해.
RNN
Recurrent Neural Network. 예전에 많이 쓴 AI 방식. 이전 내용을 조금씩 요약해가며 기억해서, 오래된 건 잊어버려.
RoPE
Rotary Position Embedding. AI한테 "이 단어가 몇 번째 위치야" 하고 알려주는 방법.
스파스 (Sparse)
"듬성듬성"이라는 뜻. 모든 걸 보지 않고 중요한 것만 골라 보는 방식.
토큰 1억 개
한국어 소설 약 333권 분량. 약 7,500만 단어. 평생 읽을 책의 상당 부분!
엔드-투-엔드 (End-to-End)
처음부터 끝까지 한 번에 통째로 학습하는 것. 부품이 따로따로 학습하지 않아.
NIAH 테스트
Needle-In-A-Haystack. 거대한 텍스트 사이에 숨긴 정보를 찾을 수 있는지 테스트. "건초더미에서 바늘 찾기".
멀티홉 추론
A → B → C 처럼 여러 단계를 거쳐 답에 도달하는 추론. "영희가 왜 슬프냐"는 여러 챕터를 연결해야 알 수 있어.
선형 복잡도 O(L)
입력이 2배 늘면 처리 시간도 2배만 늘어나는 것. 기존 어텐션은 4배 늘어났어!
