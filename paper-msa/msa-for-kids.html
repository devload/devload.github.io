Chapter 2

기존에는 어떻게 해결했어? 🔧

AI 기억력 문제를 해결하려는 시도가 여러 가지 있었는데, 모두 단점이 있었어.

기존 방법 1: 전부 다 기억하기 (Full Attention)

📚 비유: 도서관 전체 훑기

질문에 답하려고 도서관에 있는 책을 전부 다 펼쳐놓는 거야.

✅ 장점: 아무것도 빠뜨리지 않아!

❌ 단점: 책이 많아질수록 시간이 제곱으로 늘어나! 책 10배 많아지면 시간은 100배!

기존 방법 2: RAG (외부 검색)

🔍 비유: 구글 검색 + AI

AI가 모르면 인터넷에서 찾아보는 것처럼, 외부 데이터베이스에서 관련 내용을 검색해서 가져와.

✅ 장점: 큰 데이터도 다룰 수 있어!

❌ 단점: AI와 검색 시스템이 따로따로라서 "함께 공부"를 못 해. 복잡한 추론은 어려워.

기존 방법 3: RNN (메모리 압축)

🧃 비유: 주스로 압축하기

사과 10개를 사과즙 1컵으로 만드는 것처럼, 읽은 내용을 작은 "상태(state)"로 압축해.

✅ 장점: 공간을 적게 차지해!

❌ 단점: 압축하면 중요한 정보가 사라져버려. 사과 원본의 식감은 주스에 없잖아!

세 방법 비교: "불가능의 삼각형"

연구팀은 기존 방법들이 세 가지를 동시에 만족하지 못한다고 했어. 이걸 "불가능의 삼각형"이라고 불러.

MSA는 세 가지를 동시에 해결해! 🎉

방법 확장성 정밀도 효율성
Full Attention ❌ 매우 느려짐 ✅ 높음 ❌ 낭비
RAG ✅ 확장 가능 ⚠️ 보통 ⚠️ 복잡
RNN ⚠️ 보통 ❌ 압축손실 ✅ 빠름
🏆 MSA ✅ 선형 확장 ✅ <9% 손실 ✅ 효율적
Chapter 3

MSA가 뭔데? 🚀

🧠

MSA = Memory Sparse Attention (메모리 희소 어텐션)

이름이 어렵지? 쪼개서 이해해보자!

📖 이름 해석
Memory

= 기억. AI가 읽은 내용을 기억해두는 공간

Sparse

= 듬성듬성, 필요한 것만. 전부 다 보지 않아!

Attention

= 집중. 지금 질문과 관련 있는 것에만 집중!

핵심 아이디어: 똑똑한 사서

📚 비유: 슈퍼 사서 선생님

도서관에 책이 100만 권 있어. 네가 질문하면:


😩 기존 방법: 100만 권 전부 다 펼쳐서 읽어 → 하루 종일 걸려


😊 MSA 방법: 똑똑한 사서가 "이 질문이면 3층 과학관 5번 선반이랑 1층 역사관 12번 선반 책만 봐!"라고 알려줘. 그 책들만 골라서 읽어. → 5분이면 끝!


이 "똑똑한 사서"가 바로 MSA의 핵심이야! 🎯

어떻게 필요한 것만 고를 수 있어?

MSA는 2단계 선별 과정을 써.

1
대주제 선별 (Topic-level Screening)

모든 문서를 대주제로 분류해. "과학", "역사", "수학"처럼. 질문이 "물리법칙에 대해 알려줘"이면 과학 관련 문서들만 후보로 남겨.

2
세부 내용 선별 (Fine-grained Matching)

후보 문서들 중에서 진짜 필요한 부분만 골라내. "물리법칙"이면 만유인력의 법칙이 있는 단락을 딱 찝어서 가져와.

🎯 비유: 도서관에서 책 찾기

1단계: "과학 관련 책이 있는 층/서가로만 가" (대분류)

2단계: "그 중에서 '중력'이 설명된 페이지를 펴" (세부 검색)

두 단계로 나누니까 훨씬 빠르고 정확해! 🎉

Chapter 4

MSA의 4가지 비밀 무기 ⚔️

MSA가 기적같은 성능을 내는 데는 4가지 핵심 기술이 있어. 마치 슈퍼히어로의 4가지 능력 같은 거야!

🎯
① 메모리 희소 어텐션 (Sparse Attention)
전체 기억 중 필요한 것만 골라서 집중해. 도서관 100만 권 중 딱 필요한 책 몇 권만 꺼내는 것처럼. 덕분에 계산량이 엄청 줄어!
선형 복잡도 O(L) 2단계 라우팅
📍
② 문서별 위치 표시 (Document-wise RoPE)
책마다 "이 문장은 책 안에서 몇 번째야"를 따로 기억해. 책의 순서가 섞여도 각 책 안의 내용은 정확하게 이해할 수 있어!
위치 혼동 방지 100M 토큰 확장
⚡
③ KV 캐시 압축 + 병렬 처리 (KV Compression)
기억을 효율적으로 저장하고 여러 컴퓨터가 나눠서 처리해. GPU 메모리가 부족해도 CPU 메모리를 창고처럼 써서 해결!
GPU 2개로 1억 토큰 빠른 불러오기
🔄
④ 메모리 인터리브 (Memory Interleave)
여러 문서에 걸쳐 있는 복잡한 추론을 위해 "검색 → 확장 → 생성"을 여러 번 반복해. 미스터리 소설처럼 단서를 하나씩 모아서 답을 찾아가!
멀티홉 추론 단서 수집

② 문서별 위치 표시, 이게 왜 중요해?

📖 비유: 책 페이지 번호

책 세 권을 섞어서 한 권으로 만들었어. 페이지 번호가 이렇게 되면?

책A(1~300) + 책B(301~600) + 책C(601~900) → 전체 페이지 900장

근데 이러면 AI가 "301페이지"가 책B의 첫 페이지인지 모르고 혼란스러워해!


✅ MSA의 방법: 각 책의 페이지를 따로따로 기억해!

책A: 1~300쪽, 책B: 1~300쪽, 책C: 1~300쪽

책이 어느 위치에 저장되든, 각 책 안의 순서는 완벽하게 유지돼! 🎉

🔢 이게 왜 대단한 거야?

기존 AI는 학습할 때 64,000 토큰(약 책 0.5권)으로 배웠어도, MSA는 1억 토큰(책 700권!)까지 확장이 돼. 배울 때보다 1,500배 많은 내용도 처리 가능!

④ 메모리 인터리브가 뭔지 더 자세히 알고 싶어!

🕵️ 비유: 탐정 추리

복잡한 미스터리를 풀 때 탐정은 이렇게 해:

🔍
단서 검색: "이 사건의 단서를 모아봐"
💡
가설 수립: "범인은 A일 수도 있어. 그러면 B의 증언을 더 찾아야겠어"
🔍
추가 검색: "B의 증언이랑 관련된 다른 단서 더 찾기"
🎯
최종 답변: "모든 단서를 종합하면 범인은 C야!"

MSA가 이렇게 여러 번 검색하고 추론을 반복해서 복잡한 질문도 완벽하게 답할 수 있어!

Bonus

어려운 용어 사전 📖

🔤 토큰 (Token)

AI가 글을 처리하는 기본 단위. 한국어 한 글자는 보통 토큰 1~3개. "안녕"은 약 2토큰이야.

🧠 어텐션 (Attention)

"집중"이라는 뜻. AI가 글을 읽을 때 어떤 부분에 집중할지 결정하는 메커니즘. 인간이 중요한 부분에 형광펜 치는 것과 비슷해.

💾 KV 캐시 (KV Cache)

AI가 읽은 내용을 임시로 저장해두는 공간. K(Key)는 색인, V(Value)는 실제 내용이야. 마치 도서관 카드목록 + 책 내용 같은 것.

📍 RoPE (Rotary Position Embedding)

AI에게 "이 단어가 문장에서 몇 번째야"를 알려주는 방법. 위치 정보를 각도(회전)로 표현해서 긴 문장에서도 잘 작동해.

🔍 RAG (Retrieval-Augmented Generation)

AI가 모르는 것이 있으면 외부 데이터베이스에서 검색해서 가져오는 방법. 구글에서 검색해서 답변하는 것과 비슷해.

🌾 NIAH (Needle In A Haystack)

건초더미에서 바늘 찾기. 엄청 긴 글 중간에 숨겨진 정보를 찾을 수 있는지 테스트하는 AI 평가 방법이야.

⚡ GPU (Graphics Processing Unit)

게임용 그래픽카드인데, AI 계산에도 엄청 많이 쓰여. 여러 계산을 동시에 처리하는 능력이 뛰어나.

🔗 멀티홉 추론 (Multi-hop Reasoning)

여러 단계를 거쳐서 답을 찾는 것. "A가 B의 부모고, B가 C의 부모이면 A와 C의 관계는?" 같은 복잡한 추론이야.

📐 선형 복잡도 O(L)

데이터가 L배 많아지면 계산 시간도 L배만 늘어나는 것. 반대로 O(L²)이면 데이터가 10배 많으면 시간은 100배 걸려. MSA는 O(L)이라서 효율적!
