Chapter 1

기존 방법의 문제점

미래를 예측하는 건 정말 어려운 일이에요. 지금까지 어떤 방법들이 있었고, 뭐가 문제였는지 볼까요?

방법 1: 통계 방법 (ARIMA 등)

쉬운 비유
공식 하나로 맞추기 = 자 하나로 그림 그리기
직선이나 간단한 패턴은 잘 그리지만, 복잡한 곡선이나 갑자기 변하는 모양은 못 그려요.

장점: 간단하고 빠르다
단점: 복잡한 패턴은 못 잡는다. 매번 데이터마다 새로 설정해야 한다.

방법 2: 딥러닝 모델 (DeepAR, PatchTST 등)

쉬운 비유
전용 AI = 과목별 과외 선생님
수학 과외 선생님은 수학만 가르칠 수 있어요. 영어를 가르치려면 다른 선생님을 다시 구해야 해요. 매번 새로운 데이터마다 처음부터 학습시켜야 해요.

장점: 복잡한 패턴도 잘 잡는다
단점: 새 데이터마다 처음부터 학습 필요. 시간과 비용이 많이 든다.

방법 3: LLM 활용 (GPT로 예측)

쉬운 비유
글짓기 선생님에게 수학 문제 풀게 하기
GPT는 글을 잘 쓰지만, 숫자 패턴을 이해하는 데는 효율적이지 않아요. 숫자를 텍스트로 바꿔야 해서 느리고, 모델도 너무 커요.

장점: 이미 만들어져 있다
단점: 숫자를 글자로 변환해야 해서 비효율적. 모델이 수백~수천억 파라미터로 너무 크다.

그래서 TimesFM이 필요해!

통계 방법처럼 빠르고

딥러닝처럼 똑똑하고

LLM처럼 범용적이지만

LLM보다 훨씬 작고 효율적인 모델!

Chapter 2

TimesFM의 핵심 아이디어 3가지

핵심 1: 패칭 (Patching) - 숫자를 묶어서 보기

시계열 숫자를 하나씩 보면 너무 느려요. TimesFM은 숫자들을 32개씩 묶어서 한 번에 봐요. 이 묶음을 "패치(Patch)"라고 불러요.

쉬운 비유
패칭 = 책 읽기 방법의 차이!
한 글자씩 읽기: ㅎ-ㅏ-ㄴ-ㄱ-ㅜ-ㄱ-... (너무 느려! 😩)
단어씩 읽기: 한국-사람-이라면-... (빠르고 의미도 잘 파악! 😊)

TimesFM은 숫자 32개를 한 단어처럼 묶어서 읽어요!
📊 숫자 128개 → 📦 32개씩 묶기 → 🧩 패치 4개

비대칭 패칭 - 진짜 똑똑한 부분!

TimesFM의 비밀 무기는 "입력은 작게, 출력은 크게" 하는 거예요.

📥 입력 패치

  • 크기: 32개 숫자
  • 과거 데이터를 읽을 때
  • 세밀하게 패턴을 파악

📤 출력 패치

  • 크기: 128개 숫자
  • 미래를 예측할 때
  • 한 번에 많이 예측!
왜 이렇게 할까?
시험 볼 때 문제를 한 줄씩 꼼꼼히 읽지만, 답은 한 번에 쭉 쓰잖아요?
입력은 꼼꼼히(32개), 출력은 시원하게(128개) — 이래야 빠르고 정확해요!

만약 128개를 예측해야 하는데 32개씩 출력하면 4번 반복해야 하지만, 128개씩 출력하면 1번이면 돼요. 반복할수록 오차가 쌓이니까, 적게 반복할수록 좋아요!

핵심 2: Decoder-Only Transformer - ChatGPT랑 같은 구조!

TimesFM은 ChatGPT와 같은 종류의 뇌 구조를 써요. 다만 글자 대신 숫자 패치를 처리해요.

쉬운 비유
Transformer = 교실의 토론!

교실에 학생(패치)들이 앉아 있어요. 선생님이 "자, 앞에 있는 친구들의 발표를 듣고 다음 내용을 예측해봐!" 하면, 맨 뒤 학생이 앞의 모든 발표를 참고해서 답을 말해요.

이게 바로 "Causal Self-Attention" — 각 패치는 자기 앞에 있는 패치만 참고할 수 있어요. (뒤는 아직 미래니까 못 봐요!)

핵심 3: 제로샷 (Zero-Shot) - 한 번 배우면 다 할 수 있어!

쉬운 비유
제로샷 = 만능 운동선수!

보통 선수는 축구만, 농구만 연습해요. 하지만 만능 운동선수는 기본기가 탄탄해서 처음 보는 스포츠도 곧잘 해요.

TimesFM은 1000억 개의 다양한 숫자 패턴을 배워서, 처음 보는 데이터도 추가 학습 없이 바로 예측할 수 있어요!
Chapter 3

TimesFM은 어떻게 작동하나요?

과거 데이터가 들어오면 미래 예측이 나올 때까지, 단계별로 따라가 볼게요!

전체 흐름도

📊 입력: 과거 시계열 데이터
[10, 12, 15, 13, 18, 20, 22, ...]
▼
✂️ Step 1: 패칭 (32개씩 자르기)
128개 숫자 → 패치 4개
▼
🔄 Step 2: Residual Block (MLP)
각 패치를 1280차원 벡터로 변환
▼
📍 Step 3: 위치 인코딩 추가
"너는 1번째, 너는 2번째..." 순서 알려주기
▼
🧠 Step 4: Transformer 20층 통과!
각 패치가 앞의 패치들을 참고하며 패턴 파악
▼
🔄 Step 5: 출력 Residual Block
1280차원 → 128개 숫자로 변환
▼
🎯 출력: 미래 128개 시점 예측!
[23, 25, 24, 27, 30, ...]

Step 1~5: 숫자로 직접 따라가 보기!

간단한 예시로 전체 과정을 숫자와 함께 따라가 볼게요. (실제로는 32개씩이지만, 이해하기 쉽게 4개씩으로 줄여서 보여줄게요!)

숫자 워크스루: 8개 숫자로 미래 4개 예측하기

Step 1: 패칭

8개 숫자를 4개씩 2묶음으로 나눠요

입력: [10, 20, 15, 25, 30, 20, 35, 40]
▼ 4개씩 자르기
패치1 = [10, 20, 15, 25]   패치2 = [30, 20, 35, 40]
▼
Step 2: Residual Block

각 패치(4개 숫자)를 AI의 언어(1280차원 벡터)로 번역

패치1 [10,20,15,25] → MLP → [0.32, -0.15, 0.87, ..., 0.44] (1280개)
패치2 [30,20,35,40] → MLP → [0.71, 0.23, -0.56, ..., 0.18] (1280개)
▼
Step 3: 위치 인코딩

각 벡터에 순서 정보를 더해요

패치1 벡터 + PE(1번째) = 토큰1
패치2 벡터 + PE(2번째) = 토큰2
▼
Step 4: Transformer 20층

토큰들이 서로 참고하며 패턴을 파악 (20번 반복!)

토큰1: "나는 혼자 생각..." → 출력1
토큰2: "토큰1을 참고하면... 올라갔다 내렸다 올라가는 패턴!" → 출력2
▼
Step 5: 출력 Residual Block

마지막 토큰(출력2)을 다시 숫자로 변환

출력2 [0.91, -0.33, ...] → MLP → [42, 38, 45, 50]
🎯
예측 결과: [42, 38, 45, 50] ← 미래 4개 시점!

실제 스케일에서는?

위의 예시는 이해를 위해 줄인 거고, 실제로는:

# 실제: 128개 숫자를 32개씩 4묶음으로
시계열 = [10, 12, 15, ..., 22] # 128개

패치1 = [10, 12, 15, ..., 18] # 1~32번째
패치2 = [20, 22, 19, ..., 25] # 33~64번째
패치3 = [27, 30, 28, ..., 33] # 65~96번째
패치4 = [35, 38, 36, ..., 40] # 97~128번째

# 마지막 패치4의 출력 → 미래 128개 시점 예측!

Step 2: Residual Block - 숫자를 AI가 이해하는 언어로

패치(32개 숫자)를 AI가 이해하는 1280차원 벡터로 바꿔요.

쉬운 비유
Residual Block = 번역기!
우리가 쓰는 한국어(숫자 32개)를 AI의 언어(1280차원 벡터)로 번역해요.

"Residual"이 뭐냐면: 번역할 때 원본도 함께 보관하는 거예요. 마치 시험지에 답을 쓸 때 문제도 같이 보면서 쓰는 것처럼! 이래야 정보를 잃어버리지 않아요.

Step 3: 위치 인코딩 - 순서표 붙이기

쉬운 비유
위치 인코딩 = 줄 서기 번호표!
놀이공원에서 줄을 서면 번호표를 받잖아요? AI도 각 패치에 "너는 1번째, 너는 2번째"라고 번호를 붙여야 순서를 기억해요. 시계열에서 순서는 매우 중요하거든요!

Step 4: Transformer 20층 - 핵심 두뇌!

이 부분이 TimesFM의 진짜 두뇌예요. 20개 층을 겹겹이 쌓아서 패턴을 깊이 이해해요.

각 층에서는 두 가지 작업을 해요:

1) Causal Self-Attention (인과적 셀프 어텐션)

쉬운 비유
어텐션 = 토론 시간!

패치들이 서로 대화하는 시간이에요. 하지만 규칙이 있어요:
- 패치 1은 혼자만 생각해요 (앞에 아무도 없으니까)
- 패치 2는 패치 1의 말을 참고해요
- 패치 3은 패치 1, 2의 말을 참고해요
- 패치 4는 패치 1, 2, 3의 말을 모두 참고해요

절대 뒤를 보면 안 돼요! 미래 정보를 미리 보면 컨닝이니까요. 이게 "Causal(인과적)"이라는 뜻이에요.

어텐션 마스크 - 누가 누구를 볼 수 있을까?

4개 패치가 있을 때, 누가 누구를 참고할 수 있는지 표로 볼게요:

패치1 보기 패치2 보기 패치3 보기 패치4 보기
패치1 O 볼 수 있음 X 미래! X 미래! X 미래!
패치2 O 볼 수 있음 O 볼 수 있음 X 미래! X 미래!
패치3 O 볼 수 있음 O 볼 수 있음 O 볼 수 있음 X 미래!
패치4 O 볼 수 있음 O 볼 수 있음 O 볼 수 있음 O 볼 수 있음

왼쪽 아래 초록색 삼각형 모양이 보이죠? 이걸 "Causal Mask (인과적 마스크)"라고 불러요. 미래(오른쪽 위)는 절대 볼 수 없어요!

16개 어텐션 헤드를 사용해요. 이건 16명의 전문가가 각각 다른 관점으로 패턴을 분석하는 거예요.

쉬운 비유
16개 헤드 = 16명의 탐정!
탐정 1: "이 숫자들, 점점 올라가는 추세야!" (트렌드 전문)
탐정 2: "매주 월요일마다 숫자가 올라가네!" (계절성 전문)
탐정 3: "갑자기 튀는 값이 있어!" (이상치 전문)
...
16명이 각자 다른 패턴을 찾아내고, 합쳐서 종합 판단해요!

2) Feed-Forward Network (피드포워드 네트워크)

어텐션에서 수집한 정보를 깊이 처리하는 단계예요. 간단히 말해 "생각하는 시간"이에요.

Step 5: 출력 - 미래 예측!

마지막 패치의 출력(1280차원 벡터)을 다시 128개 숫자로 변환하면, 그게 바로 미래 예측이에요!

더 먼 미래가 필요하면? - 오토리그레시브 디코딩

쉬운 비유
오토리그레시브 = 이어달리기!

128개를 예측했으면, 그걸 다시 입력에 붙여서 그 다음 128개를 예측해요.

과거 데이터 [1~128] → 예측 [129~256]
과거 + 예측 [1~256] → 예측 [257~384]
과거 + 예측 [1~384] → 예측 [385~512]

이어달리기처럼 예측한 결과를 바통 삼아 계속 이어가요!

쉬운 비유
합성 데이터 = AI용 수학 문제집!
실제 시험(실제 데이터)만으로는 연습이 부족해서, 선생님이 다양한 유형의 연습문제를 만들어 줬어요. 4가지 유형을 섞어서 만들어요:

트렌드

🔄

컨텍스트 마스킹 - 가변 길이 학습의 비밀

쉬운 비유
마스킹 = 문제의 일부를 가리기!

시험 공부할 때 문제를 전부 보고 풀기도 하고, 앞부분을 가리고 뒷부분만 보고 풀기도 해요. 이렇게 다양한 길이의 입력에 익숙해지면, 실전에서 어떤 길이의 데이터가 와도 당황하지 않아요!

학습할 때마다 랜덤으로 앞부분을 가려서, 1개부터 최대 512개까지 모든 길이에 대비시켜요.

손실 함수 = 채점 기준

AI가 얼마나 잘 맞췄는지 점수를 매기는 기준이에요.

쉬운 비유
MSE = 오답노트의 점수!

AI가 "25"라고 예측했는데 정답이 "20"이면, 차이는 5이고, 이걸 제곱하면 25점 감점!
(25-20)² = 25

모든 예측에 대해 이런 점수를 매기고 평균 내요. 이 점수가 작을수록 AI가 잘 맞춘 거예요!
학습하면서 이 점수를 계속 줄여나가요.

LLMTime(GPT-3) vs TimesFM

왜 GPT-3이랑 비교해?
GPT-4나 GPT-4o가 아니라 GPT-3인 이유가 있어요. LLMTime이라는 논문(2023)이 "LLM으로도 시계열 예측을 할 수 있다!"를 처음 보여줬는데, 그 논문이 GPT-3를 사용했어요. TimesFM 논문 시점(2023.10)에 "LLM 기반 시계열 예측"의 대표 연구가 바로 이 LLMTime이었기 때문에, "우리 방식이 LLM 방식보다 낫다"를 증명하기 위한 비교예요.

LLMTime (GPT-3 기반)

  • 파라미터: 1750억 개
  • 숫자를 글자로 변환해서 처리
  • 시계열 전용이 아님
  • 느리고 비싸다

TimesFM

  • 파라미터: 2억 개 (875배 작음!)
  • 숫자를 숫자 그대로 처리
  • 시계열 전용 설계
  • 빠르고 효율적

TimesFM이 GPT-3 기반 LLMTime보다 875배 작으면서도 10% 이상 더 정확해요!

스케일링 법칙 - 크면 클수록 좋아!

쉬운 비유
스케일링 = 공부를 더 하면 성적이 오르는 것!

17M(1700만) → 70M(7000만) → 200M(2억) 파라미터로 키울수록 성능이 꾸준히 올라갔어요.
LLM(ChatGPT 등)에서도 나타나는 현상이 시계열 모델에서도 확인된 거예요!

Glossary

용어 사전

시계열 (Time Series)
시간 순서대로 기록된 숫자 데이터. 매일 기온, 매시간 교통량 등.
패칭 (Patching)
시계열 숫자를 일정 크기(32개)로 묶는 것. 단어를 만드는 것과 비슷.
Transformer
ChatGPT에도 쓰이는 AI 구조. 입력들 사이의 관계를 파악하는 데 뛰어남.
Decoder-Only
Transformer의 한 종류. 앞의 내용만 보고 다음을 예측하는 구조. GPT와 같은 방식.
Self-Attention
각 패치가 다른 패치들과 "대화"하면서 중요한 패턴을 찾는 메커니즘.
Causal (인과적)
미래 정보를 참고하지 않는 규칙. 현재와 과거만 보고 예측해야 해요.
Residual Block
원래 값을 보존하면서 변환하는 구조. 정보 손실을 막는 안전장치.
위치 인코딩 (Positional Encoding)
각 패치에 순서 정보를 더해주는 것. "1번째, 2번째..."를 알려줘요.
제로샷 (Zero-Shot)
추가 학습 없이 바로 예측하는 능력. 처음 보는 문제도 풀 수 있어요.
파라미터 (Parameters)
AI의 "뇌세포" 같은 것. 학습하면서 조절되는 숫자들. 많을수록 복잡한 패턴을 배울 수 있어요.
MSE (Mean Squared Error)
예측값과 실제값의 차이를 제곱해서 평균 낸 것. 작을수록 잘 맞춘 거예요.
오토리그레시브 (Autoregressive)
자기가 예측한 결과를 다시 입력으로 사용하는 방식. 이어달리기처럼 계속 이어감.
양자일 (Quantile)
"정확한 값은 모르지만 이 범위 안에 있을 거야"라고 범위로 예측하는 것.
컨텍스트 길이 (Context Length)
AI가 한 번에 볼 수 있는 과거 데이터의 최대 길이. v2.5에서는 16,384까지!
호라이즌 (Horizon)
얼마나 먼 미래까지 예측할지. "호라이즌 12" = 12 시점 뒤까지 예측.
파운데이션 모델 (Foundation Model)
대규모 데이터로 사전학습하여 다양한 작업에 범용적으로 사용 가능한 AI 모델.
